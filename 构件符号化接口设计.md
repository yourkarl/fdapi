# TileLayer 构件符号化接口设计

> 面向 BIM 构件（Actor）的符号化能力设计。基于底层协议 `AddSymbols` / `RemoveSymbols`，在 `fdapi.tileLayer` 上新增「设置 / 查询 / 恢复」三个方法。命名与参数风格对齐现有 `highlightActorsWithColor`（`data: object|array`、`id`、`objectIds`、`Color`）。

---

## 方法列表

| 方法 | 说明 |
|------|------|
| [`addSymbols`](#addSymbols) | 对构件进行符号化设置（着色） |
| [`getSymbols`](#getSymbols) | 查询构件当前的符号化状态 |
| [`removeSymbols`](#removeSymbols) | 恢复（移除）构件的符号化状态 |
| [`clearSymbols`](#clearSymbols) | 清空图层的全部符号化 |

---

## 方法（Methods）

### `addSymbols(data, fn)` {#addSymbols}

对 TileLayer 下的 BIM 构件（Actor）进行符号化设置。支持一次对不同构件应用不同的样式与颜色，同一图层可按多组 `symbols` 分组着色。

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|:----:|--------|------|
| `data` | `object \| array` | 是 | - | 数据对象（或数组，批量对多个图层符号化），每个对象支持以下属性： |
| `fn` | `function` | 否 | - | 可选的回调函数，请参考[二次开发：异步接口调用方式](/docs/tutorials/async-call) |

> **`data` 对象属性：**

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|:----:|--------|------|
| `id` | `string` | 是 | - | TileLayer 的 ID |
| `symbols` | `array` | 是 | - | 符号化分组数组，每组对一批构件应用同一样式，结构见下 |

> **`symbols` 元素属性：**

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|:----:|--------|------|
| `objectIds` | `array` | 是 | - | 该分组作用的构件 ObjectID 列表 |
| `style` | [`SymbolStyle`](/docs/api/types#symbolstyle) | 否 | `1` | 符号化样式类型，取值范围：整数枚举；`1` 纯色填充（其余样式见 `SymbolStyle`） |
| `color` | [`Color`](/docs/api/types#color) | 否 | `[1,1,1,1]` | 符号化颜色，支持四种格式，[取值示例](/docs/tutorials/color) |

**返回：** 无返回数据；异步方法，可 `await` 等待执行完成，或在回调函数 `fn` 中处理。

> 示例：对构件符号化：AddSymbols

```js
// 对同一图层内的两组构件分别着色
await fdapi.tileLayer.addSymbols({
  id: 'FB92292E4EB1B0C5F415A48C2F53B0D3',
  symbols: [
    { style: 1, color: [1, 0, 0, 1], objectIds: ['1', '2', '3'] }, // 红
    { style: 1, color: [0, 1, 0, 1], objectIds: ['4', '5'] },      // 绿
  ],
});
```

> 对应底层协议：

```json
{
  "data": [{
    "id": "FB92292E4EB1B0C5F415A48C2F53B0D3",
    "symbols": [{
      "style": 1, "color": [1, 1, 1, 1], "objectIds": ["1", "2", "3", "4", "5"]
    }]
  }],
  "command": "AddSymbols"
}
```

### `getSymbols(data, fn)` {#getSymbols}

查询 TileLayer 下构件当前的符号化状态。不传 `objectIds` 时返回该图层内所有已符号化的构件。

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|:----:|--------|------|
| `data` | `object \| array` | 是 | - | 查询条件对象，支持以下属性： |
| `fn` | `function` | 否 | - | 可选的回调函数，请参考[二次开发：异步接口调用方式](/docs/tutorials/async-call) |

> **`data` 对象属性：**

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|:----:|--------|------|
| `id` | `string` | 是 | - | TileLayer 的 ID |
| `objectIds` | `array` | 否 | 全部 | 需查询的构件 ObjectID 列表；省略则返回该图层所有已符号化构件 |

**返回：** 异步方法，查询结果通过回调函数 `fn` 返回（也可 `await` 获取），具体数据结构见示例。

> **返回数据结构**

```js
{
  "id": "FB92292E4EB1B0C5F415A48C2F53B0D3",
  "symbols": [
    { "objectId": "1", "symbolized": true,  "style": 1, "color": [1, 0, 0, 1] },
    { "objectId": "4", "symbolized": true,  "style": 1, "color": [0, 1, 0, 1] },
    { "objectId": "9", "symbolized": false }   // 未符号化的构件
  ]
}
```

> 示例：查询符号化状态：GetSymbols

```js
let res = await fdapi.tileLayer.getSymbols({
  id: 'FB92292E4EB1B0C5F415A48C2F53B0D3',
  objectIds: ['1', '4', '9'],
});
log('符号化构件数量: ' + res.symbols.filter(s => s.symbolized).length);
```

### `removeSymbols(ids, fn)` {#removeSymbols}

恢复（移除）图层的符号化状态，使其还原为原始外观。清除指定 TileLayer 图层内所有构件的符号化；不传 `ids` 时恢复**当前场景所有图层**的符号化。参数与 [`clearSymbols`](#clearSymbols) 保持一致。

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|:----:|--------|------|
| `ids` | `string \| array` | 否 | 全部图层 | TileLayer 的 ID（单个字符串或 ID 数组）；省略则恢复所有图层的符号化 |
| `fn` | `function` | 否 | - | 可选的回调函数，请参考[二次开发：异步接口调用方式](/docs/tutorials/async-call) |

**返回：** 无返回数据；异步方法，可 `await` 等待执行完成，或在回调函数 `fn` 中处理。

> 示例：恢复符号化状态：RemoveSymbols

```js
// 恢复单个图层的符号化
await fdapi.tileLayer.removeSymbols('FB92292E4EB1B0C5F415A48C2F53B0D3');

// 恢复多个图层
await fdapi.tileLayer.removeSymbols(['图层ID1', '图层ID2']);

// 恢复当前场景所有图层的符号化（不传参数）
await fdapi.tileLayer.removeSymbols();
```

> 对应底层协议（按图层下发，`objectIds` 省略表示该图层全部构件）：

```json
{
  "data": [{ "id": "FB92292E4EB1B0C5F415A48C2F53B0D3" }],
  "command": "RemoveSymbols"
}
```

### `clearSymbols(ids, fn)` {#clearSymbols}

清空符号化。清除指定 TileLayer 图层内所有构件的符号化状态；不传 `ids` 时清空**当前场景所有图层**的符号化。对标 `unHighlightAllActors`，用于一键重置。

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|:----:|--------|------|
| `ids` | `string \| array` | 否 | 全部图层 | TileLayer 的 ID（单个字符串或 ID 数组）；省略则清空所有图层的符号化 |
| `fn` | `function` | 否 | - | 可选的回调函数，请参考[二次开发：异步接口调用方式](/docs/tutorials/async-call) |

**返回：** 无返回数据；异步方法，可 `await` 等待执行完成，或在回调函数 `fn` 中处理。

> 示例：清空符号化：ClearSymbols

```js
// 清空单个图层的全部符号化
await fdapi.tileLayer.clearSymbols('FB92292E4EB1B0C5F415A48C2F53B0D3');

// 清空多个图层
await fdapi.tileLayer.clearSymbols(['图层ID1', '图层ID2']);

// 清空当前场景所有图层的符号化（不传参数）
await fdapi.tileLayer.clearSymbols();
```

> `clearSymbols` 与 [`removeSymbols`](#removeSymbols) 参数、行为完全一致，互为别名，可任选其一使用。

## 设计要点

- **贴合底层协议**：`addSymbols` / `removeSymbols` 与 `AddSymbols` / `RemoveSymbols` 一一对应，`data` 结构（`id` + `symbols`/`objectIds`）直接透传，SDK 只做入参归一化，降低适配成本。
- **对齐现有约定**：`data` 支持 `object | array`、`id` 为图层 ID、`objectIds` 为构件 ID、颜色复用 [`Color`](/docs/api/types#color)，与 `highlightActorsWithColor` 完全一致，学习成本低。
- **易用的缺省**：查询 / 恢复的 `objectIds` 可省略表示「全部」；`style`、`color` 有默认值，最简调用只需 `id` + `objectIds`。
- **符号化 vs 高亮**：符号化是持久的构件着色（专题着色 / 状态区分），高亮是临时选中效果，二者独立，命名上用 `Symbols` 与 `Highlight` 区分，避免混淆。

---

## 待确认

1. **`style` 枚举取值**：底层示例为 `style:1`，暂按「1 = 纯色填充」占位。需补充符号化支持的完整样式清单（如纯色 / 半透明 / 线框 / 贴图 / 渐变）以定义 `SymbolStyle` 枚举。
2. **是否需要 `updateSymbols`**：若同一构件重复 `addSymbols` 即为覆盖更新，则无需单独的修改方法；若底层区分新增与修改，再补 `updateSymbols`。
