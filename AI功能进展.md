# DTS 二次开发文档项目 · AI 功能进展

> 汇总当前 fdapi 文档工程中与 AI 相关的功能与进展，供同步到在线文档。

## 一、AI 代码助手（调试台内置）

文档站的调试台 / 示例台内置了 **DTS SDK AI 代码助手**（`src/components/AIChat.js`），用户用自然语言描述想要的三维场景效果，助手直接输出可运行的 fdapi JavaScript 代码。例如输入「飞行到腾讯滨海大厦并俯视」「添加一个蓝色发光标注点」「开启降雨+雾效」「获取所有相机动画列表」即可得到对应代码。

**多 AI 提供商，浏览器直连，Key 本地存储：**

- Claude：Haiku 4.5（快速）/ Sonnet 4.6（均衡）/ Opus 4.8（强力）
- ChatGPT：GPT-4.1 / GPT-4o / o3 / o4-mini 等
- DeepSeek：V4-Flash / V4-Pro / Chat / R1（推理）
- 通义千问：Turbo / Plus / Max / Qwen3-235B / QwQ / Qwen-Long
- 支持自定义模型 ID

各厂商 API 由浏览器直连，API Key 仅存本地（localStorage），不经过服务端。

## 二、防「臆造 API」三重机制（准确度保障）

AI 生成代码最大的风险是编造不存在的接口或参数，本项目为此做了三层约束，显著降低幻觉：

1. **按需注入真实签名**：中文意图 → fdapi 命名空间映射（NS_ALIASES，如「下雨/下雪/雾」→ `weather`、「单体化/高亮」→ `tileLayer`），命中后把相关命名空间的真实方法签名注入提示词，要求「严格按参数顺序、数量与类型生成，禁止臆造参数」。
2. **fdapi 白名单校验**：以文档全量 API（`api-completions.js`，权威）加真实示例为基准构建白名单，凡有文档的方法一律放行。
3. **生成后拦截**：`findInvalidApis` 扫描生成结果，若包含文档中不存在的 fdapi 调用，直接拦截该响应并提示用户换描述重发，避免把臆造接口交付给用户。

系统提示词还约束：直接输出代码、严格遵循签名、不确定的参数以注释标注 TODO 而非编造。

## 三、面向 AI 的「业务场景 Skill」语义层

全部 **81 个 API 类**文档均新增「## 业务场景 Skill」段，明确「面向 AI 与业务人员」，统一结构：

- **功能介绍**：对象在数字孪生业务中的定位与典型用法
- **别名 / 不同行业叫法**：便于 AI 从行业口语映射到具体接口
- **适用行业**：覆盖的行业清单
- **使用场景**：可直接检索的典型业务用法
- **注意事项**：性能、坐标系、授权等要点

该层为 RAG / Agent 检索与行业问答提供了结构化语义元数据。近期结合行业方案（智慧交通、水利水务、智慧工地、智慧工厂、石油化工/油气田、智慧矿山、地下管网、人防应急、实景三维、CIM 基础平台等）持续扩充各类的「适用行业」与「使用场景」。

## 四、行业方案 Skill（行业需求 ↔ 接口映射）

在面向单个 API 类的「业务场景 Skill」之上，项目沉淀了一层**行业方案 Skill**：以「行业方案」侧栏专章的形式，把一个行业的典型业务功能点，系统映射到具体的 fdapi 接口，形成「功能 ↔ 接口」对照。它回答的是 AI 与业务人员最常问的一类问题——「这个行业要做 XX，用哪些接口、怎么组合」。

**统一结构：** 每篇行业方案包含 方案概述、若干「功能点 ↔ fdapi 接口 ↔ 说明」对照表（按业务子场景分组）、以及性能与实现要点。授权类接口统一指向《高级接口授权》，数据底座能力指向《平台技术优势》。

**已覆盖 17 个行业方向：**

- 城市与空间：CIM 基础平台、智慧社区 / CIM、智慧园区、实景三维 / 测绘
- 交通出行：智慧交通、轨道交通、港口航道 / 海洋
- 建筑与工程：智慧建筑、智慧工地、智慧工厂
- 资源能源：石油化工 / 油气田、智慧矿山、地下综合管线
- 水利与生态：智慧水利水务
- 公共服务：智慧校园、智慧文旅 / 景区、人防应急指挥

**对 AI 的价值：** 单类 Skill 解决「某个接口是什么、怎么用」，行业方案 Skill 解决「某个行业的成套需求如何落到接口组合」。两层结合，使 AI 助手既能生成单点代码，也能针对行业问题给出成体系的接口选型与实现思路，检索粒度从「接口级」上升到「行业方案级」。近期结合每批行业文档，持续回补各 API 类的「适用行业 / 使用场景」，两层语义相互印证。

## 五、配套数据与工程化

- `api-completions.js`：全量权威 API 签名（AI 注入与代码补全基准）
- `real-examples.js`：真实可运行示例
- `param-meta.js`：参数元数据（含必填 / 默认值）
- `check_docs_integrity.py`：文档完整性校验（UTF-8 / 无 NUL / front matter / 代码块闭合 / 链接完整），保障 AI 依赖的文档语料质量
