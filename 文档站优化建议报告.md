# DTS SDK 文档站优化建议报告

> 对标高德地图 JSAPI / Google Maps Platform 开发者文档，2026-07-02

## 总体评价

项目基础扎实：126 篇文档、19 个 API 分类、18 篇教程 + 17 个行业方案、在线调试台（真实云渲染连接 + 1600+ 示例 + AI 助手 + 参数面板 + TS 补全）、中文分词搜索、CI 完整性检查、原子写入生成管线。**调试台 + AI 助手 + 业务场景 Skill 段落甚至领先于高德/谷歌的同类能力。** 综合成熟度约 6.5~7/10，主要差距在"API 参考的严谨性"和"错误处理体系"。

---

## P0 — 直接影响开发者能否用对 API

### 1. 返回值说明系统性缺失（最大短板）
抽查发现 75+ 方法只写"返回 Promise"，不给出 resolve 的数据结构示例。对标：Google Maps 每个方法有明确的 `Returns: Type` 且类型可点击跳转；高德每个异步回调都给出结果对象字段表。

建议：为每个有返回值的方法补充「返回值」小节，含字段表 + 一段真实返回 JSON 示例。可先脚本扫描列出缺失清单，按 quickstart/camera/marker 等高频类优先补。

### 2. 零错误处理体系
全站无错误码总表、无 Promise reject 情形说明、示例几乎不含 try-catch。对标：Google Maps 有专门的错误码页（`InvalidKeyMapError` 等）；高德有 ErrorCode 对照表。

建议：
- 新增 `docs/api/error-codes.md` 全局错误码/失败情形总表；
- 每个方法的「参数」表后补充「可能的失败情形」；
- 教程中补一篇「错误处理与调试」（配合调试台的 apilog）。

### 3. 版本标注与多版本文档
API 级无 `since` / `deprecated` 标注，未启用 Docusaurus versioning。对标：高德 v1.4 / v2.0 双版本站并存；Google Maps 有 weekly/quarterly 版本渠道，每个 API 标注可用版本。

建议：短期在 front matter / 方法标题旁加 `自 v7.0` `已废弃` 徽章（可做成 MDX 组件）；SDK 出 7.2 时启用 `docusaurus docs:version 7.1`。将 `revision-history` 升级为标准 Changelog（按版本列出新增/变更/废弃 API，附迁移指引）。

## P1 — 对标竞品的体验差距

### 4. 独立「示例中心」
1652 个真实示例目前只在调试台内部导航。高德的「示例中心」是独立板块：按功能分类、缩略图墙、每个示例一键运行/看源码，是其文档最受欢迎的入口。

建议：新建 `/examples` 页面，把 `real-examples.js` 渲染成可搜索的分类卡片墙，每张卡片跳转调试台并自动载入代码；API 文档页侧边挂"相关示例"。

### 5. 图示与可视化解释
仅约 15% 教程有图。相机 pitch/yaw/roll（文档自己都承认"和场景 UI 配置不对应"）、坐标系、BBox、偏移量等空间概念全靠文字。对标：Google/高德在坐标、锚点、事件冒泡等概念均配图。

建议：优先给 camera（欧拉角示意）、coordinates（投影 vs 球面）、marker（锚点/偏移）、layer（层级）配 SVG 示意图；调试台可加"当前相机姿态可视化"小工具。

### 6. 快速参考与术语表
- 新增 API 速查表（cheat sheet）：一页列出最常用 30 个调用，可打印；
- 新增术语表：数字孪生/GIS/BIM 术语中英对照（acp、ACP 工程、云渲染实例、fdapi 等私有概念对新人是黑话）。

### 7. 上手门槛
调试台依赖手动拷贝 `ac.min.js` + 需要真实云渲染实例。对标：高德/Google 注册 key 后 CDN 一行引入即可跑。

建议：提供「演示模式」（录制的视频流回放 + mock 返回值），让没有实例的用户也能体验 API 调用闭环；hello-world 教程首屏给出最短可运行代码 + 预期效果 GIF。

### 8. 反馈闭环
每页缺"本页是否有帮助 👍👎"组件与"报告文档问题"入口（可直接链到 GitHub issue 模板并预填页面路径）。这是 Google/高德标配，也是持续改进文档的数据来源。

## P2 — 工程与安全

### 9. 安全加固（内部工具可接受，但建议处理）
- `sandbox.js:211` 用 `window.eval` 执行用户代码：属调试台本职功能，但建议改用 `new Function` + 独立 iframe sandbox，隔离对文档站自身 DOM/localStorage 的访问；
- `AIChat.js` API Key 明文存 localStorage：至少改 sessionStorage 并在 UI 明示风险，或经轻量代理转发；
- 实例密码经 URL 参数传递（`password`）：建议支持从不入 URL 的输入方式，避免留存于浏览器历史/分享链接。

### 10. 工程化
- 无 ESLint/Prettier/TS：先加 ESLint + Prettier，`src/` 渐进迁移 TS（已有 `dts-sdk.d.ts` 基础）；
- `sandbox.js`（951 行）、`AIChat.js`（867 行）拆分为子组件 + hooks；
- 生成脚本 Python/Node 双轨（`gen_param_meta.py/.js`）易漂移，建议统一到一种运行时；
- CI 可增加：`onBrokenAnchors: 'throw'`、外链检查（lychee）、Lighthouse 预算。

### 11. SEO 与分发
sitemap 已有（preset 默认）。可补充：站点 `metadata`（og:image 已有 social-card，确认图片实际存在）、`robots.txt`、结构化数据；若面向公网开发者，考虑英文 i18n（当前仅 zh-Hans）。

## 差异化优势（保持并放大）

- **AI 助手 + 业务场景 Skill 段落**：面向 AI 的文档结构（别名/行业/场景）是超前设计，建议顺势输出 `llms.txt` / MCP server，让外部 AI 编程工具直接消费文档 —— 这是高德/谷歌尚未做好的领域；
- 「在调试台运行」代码块按钮 + 分享链接，已对齐 Google 的可运行示例体验；
- 17 个行业方案教程是 GIS+BIM 垂直领域的独特资产，建议在首页做行业入口导航。

## 第二轮规范性复查（2026-07-08 补充）

已当场修复：删除无引用的垃圾文件 `static/downloads/zisai1jt`；CI 语法检查清单补入 `gen_llms_txt.py`；`@docusaurus/module-type-aliases`、`@docusaurus/types` 从 3.5.2 对齐到 ^3.10.1（lockfile 已同步，顺带清掉 786 行陈旧依赖）；新增 `.gitattributes` 统一 LF。

待处理的不规范项：

1. **social card 图片不存在**：`docusaurus.config.js` 引用 `img/dts-social-card.png`，`static/img/` 下没有该文件，分享链接时 og:image 404。需设计一张 1200×630 的卡片图放入，或暂时移除该配置。
2. **readme 与 git 实际状态矛盾**：readme 称 `ac.min.js` / `ac_conf.js`"不入库、部署时拷贝"，但两者均被 git 跟踪（ac.min.js 373KB 的 SDK 产物）。二选一：`git rm --cached` 并加入 .gitignore，或修改 readme 说法。
3. **35 篇文档无 H1**（如 tutorials/async-call.md、auth.md）：页面靠 front matter title 兜底渲染没问题，但 markdown 本身不规范，影响非 Docusaurus 场景的可读性。
4. **49 篇文档存在空 alt 图片**（`![]()`）：无障碍与 SEO 扣分，建议批量补上描述性 alt。
5. **`docs/README.md` 作为内容页命名不规范**（slug /skill-guide）：README 惯例是目录说明，建议改名 `skill-guide.md`（front matter slug 已锁定路由，改名不影响链接）。
6. **`onBrokenMarkdownLinks: 'warn'` 建议改为 `'throw'`**：与 `onBrokenLinks: 'throw'` 保持同等强度，防断链入库（改前先本地 build 确认无存量警告）。
7. CI 语法检查仍包含已封存的 `regenerate_docs.py` / `gen_sidebar.py`：无害，但与"已封存"的说法不一致，可从清单移除。

## 批量整改记录（2026-07-09）

已完成：35 篇文档补 H1（取自 front matter title，保留原换行风格）；49 篇文档 99 张空 alt 图片按「页面标题 + 就近小节」补描述性 alt；`docs/README.md` 重命名为 `skill-guide.md`（slug 未变，链接不受影响）；`onBrokenMarkdownLinks` 升级为 `'throw'`（静态断链扫描 0 处，CI 构建兜底）。

**顺带修复一个管线级 bug**：`gen_dts.py` 的方法标题正则不识别 `{#锚点}` 后缀，而 `docs/api` 的方法标题均带锚点——导致任何人跑 `npm run gen` 都会把 `dts-sdk.d.ts` 从 2762 行冲成 302 行空壳（TS 类型声明与编辑器补全静默失效）。已修正正则并重新生成（69 命名空间 / 1239 方法），新产物还顺带修正了提交版中 `BoxTrigger`/`Drone` 等命名空间大小写错误（运行时实为 `fdapi.boxTrigger`）。

提示：本次改动 90+ 文件，建议 push 前本地跑一次 `npm run build` 做最终验证（沙盒无法完成完整构建）。
