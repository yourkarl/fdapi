# 行业方案 · 智慧文旅与景区（IOC）


### 方案概述

景区智慧运营中心（IOC）以 1:1 写实三维景区为底座，融合消防、安防、能耗、广播、停车、巡更、车船等物联设备与视频、客流数据，实现「一屏管控」。依托飞渡 DTS（`fdapi`）构建文旅 / 景区数字孪生。「数字中轴 / 红色地图」等文化互动、景区平安管控均属此类。

---

## 景区三维底座

| 能力点 | fdapi 接口 | 说明 |
|------|-----------|------|
| 景区实景 / 精模 | `tileLayer` / `cesium3dTileset` / `customObject` | 1:1 写实景区建模加载与单体化 |
| 地形 / 影像底图 | `globeTerrain（球面）` / `imageryLayer（投影）+imageryLayer2（球面）` | 景区地形与卫星影像 |
| 分区 / 设施分层 | `infoTree` / `groupId` | 按景区分区、设备类型分组显隐 |

## 物联设备点位与告警

| 能力点 | fdapi 接口 | 说明 |
|------|-----------|------|
| 设备空间分布 | `marker` / `marker3d` / `customTag` | 消防、安防、水电表、广播、停车等设备点位，鼠标悬浮 / 点击弹窗（`popupURL`）展示名称 / IP / 区域 / 在线状态 |
| 设备状态着色 | `marker` / `customTag`（按状态换色） | 以不同颜色表达设备运行 / 离线 / 告警状态 |
| 告警定位（红色闪烁） | `marker` / `marker3d`（闪烁高亮）+ `highlightArea` + `camera.focus` | 设备告警时红色闪烁定位并聚焦，直至告警消除 |

## 视频、客流与实时轨迹

| 能力点 | fdapi 接口 | 说明 |
|------|-----------|------|
| 视频调阅与融合 | `videoProjection` + `marker`（摄像机点位） | 点击摄像机弹出实时视频，视频与三维场景融合 |
| 客流统计与热力 | `heatMap` / `heatMap3D` + `marker`（监测点浮窗） | 分区进 / 出客流、实时存量人数，客流密度热力 |
| 人员 / 车船实时位置 | `customObject` / `vehicle` + `camera.follow` + `moveTo` | 安保人员、车辆、船只实时位置与轨迹，支持搜索 / 定位 |
| 巡更点与巡更人员 | `marker` + `customObject` | 巡更点分布、安保人员实时位置 |
| 客流预警 / 电子围栏 | `boxTrigger` | 重点区域进出检测与人流超限告警 |

## 综合态势与导览

| 能力点 | fdapi 接口 | 说明 |
|------|-----------|------|
| 综合态势 | `heatMap` + `marker` + `highlightArea` | 客流、能耗、告警统计的一屏可视化 |
| 环境氛围 | `weather`（昼夜 / 天气 / 夜景） | 景区昼夜与天气氛围渲染 |
| 汇报导览 | `cameraTour` | 景区漫游导览与视点编排 |
| 坐标统一 | `coord` | 经纬度↔投影↔屏幕互转，设备 / 车船定位统一 |

---

## 要点

- 海量设备 / 客流点位用 `marker.add`（数组）+ `groupId` 分组与可视距离聚合；状态与告警用 `updateBegin` / `updateEnd` 批量刷新。
- 人员 / 车船轨迹按 `FrameTick` 节奏平滑驱动，配合 `camera.follow` 跟踪重点目标。
- 视频按标准协议接入并经 `videoProjection` 与三维融合；摄像机地理分布用 `marker` 表达。
