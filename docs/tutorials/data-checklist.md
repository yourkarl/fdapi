# 数据性能与体检清单

数据能进场景只是及格线，稳定流畅才是交付线。本文汇总数据侧的性能策略与上线前自查清单，通用性能优化另见[性能最佳实践](/docs/tutorials/performance)。

## 性能策略四板斧

**一、底座分层，按需加载。** 倾斜摄影、BIM、管线拆分为独立 3dt 图层，随业务视图组合显隐；全程不用的图层不要在初始化时加载。服务流式类（cesium3DTileset、影像服务）天然按视口调度，文件类图层是内存大户，数量与总量需实测预算。

**二、可视范围控制。** 点位、标注、图层普遍支持 `setViewHeightRange`（可视高度范围）：低空看设备细节、高空看聚合态势，避免万级点位在全高度渲染。markerLayer 的三级自适应机制也是同理，量级大的点位优先选它。

**三、批量更新与节流。** 所有高频更新（IoT 状态、轨迹回传）必须：`updateBegin()` / `updateEnd()` 包裹批量修改；接入层按帧合并（一帧内同一对象只保留最后一次更新）；轮询周期与业务实时性匹配，不要默认 1 秒全量刷。

**四、资源就近。** 大文件（3dt/pak/ply/shp）一律上传 CloudMaster 走 `@path:`；外部 URL 仅用于轻量图标与弹窗页面，并确认 CORS 与带宽。

## 症状 → 归因速查表

| 症状 | 最可能原因 | 检查动作 |
|------|-----------|---------|
| 加载无报错但看不到 | 位置在视野外 / assetPath 错误 | `focus` 定位到对象；核对 pak 的 assetPath |
| 加载失败 | 路径/密钥/CORS | `@path:` 路径核对；加密 3dt 补 password；DevTools 看跨域报错 |
| 整体偏移固定距离 | 坐标系不一致（GCJ-02 / 高程基准） | 见[坐标配准](/docs/tutorials/data-georeference)归因三类 |
| 边缘越来越偏 | 投影带错误 | 回数据端重投影，前端勿硬凑 |
| 模型悬空/入地 | 高程基准不一致 | 固定 z 偏移验证归因 |
| 点击构件无属性 | 单体化缺失或编码未对齐 | 确认 3dt 生产时含属性；`getActorInfoFromDB` 核对构件 ID |
| 表面闪烁（Z-fighting） | 贴地要素与地表共面 | 矢量要素抬高 0.1~0.5 m 或启用贴地模式 |
| 帧率随点位数下降 | 未分级/未限可视范围 | 换 markerLayer；设 setViewHeightRange |
| 更新时整体卡顿 | 逐条更新未批量 | updateBegin/End + 按帧合并 |
| 视频融合花屏/黑屏 | 流格式或网络 | 先用标准测试流验证通道，再查业务流 |

## 上线前数据体检清单

**配准与完整性**

- [ ] 全部数据源的 EPSG 与高程基准已记录在项目文档
- [ ] 五点校验通过（数据四角+中心与场景地物对齐）
- [ ] 构件 ID ↔ 业务编码映射表已固化为配置

**性能**

- [ ] 初始加载仅包含首屏必需图层，其余延迟/按需加载
- [ ] 万级以上点位使用 markerLayer 并配置可视高度范围
- [ ] 高频更新全部走批量 + 节流，实测峰值帧率达标
- [ ] 大文件资源全部 `@path:` 化，无外网大文件依赖

**健壮性**

- [ ] 所有 add 类调用有失败处理（Promise catch），关键图层有加载重试
- [ ] 断线重连后接入层可重建场景状态（对象清单可从业务侧重放）
- [ ] 数据服务不可用时有降级表现（占位提示而非白屏），排错路径见[排错指南](/docs/tutorials/troubleshooting)
