Cloud管理服务接口测试与二次开发说明

管理服务有2种调用方式：WebSocket和REST，两种方式发送和接收的数据格式都是JSON
    （1）在登录有效期内多次调用接口不需要重复登录，过了有效期，需要重新登录。
    （2）登录接口不能通过WebSocket调用，只能通过REST方式调用
    （3）WebSocket方式，需要先建立连接，然后才能调用
    （4）通过REST调用获取实时运行状态的接口，只能返回当前的状态，不能像WebSocket一样实时不断推送。
    （5）REST通过HTTP GET/POST/PUT。 注意：POST/PUT时Content-Type需要设置为application/json，要发送的JSON字符串设置在Body里
    （6）如果设置了实例管理密码，对于WebSocket调用，验证信息authorization直接放在要发送的JSON对象里
            对于REST GET，验证信息需要放在Request的headers里
            对于REST POST/PUT，验证信息既可以放在Request的headers里，也可以放在要发送的JSON对象里。

实例：获取一个空闲的实例 ↑↑↑

            REST: /manage/instance/free
            
接口说明：
            获取一个正在运行或者尚未启动的实例
                返回值：
                    如果当前没有空闲的，则返回错误码1: NoFreeInstance
                    如果有空闲的，则返回空闲实例的ID
                服务器查找的优先级如下：
                        1、查找已启动、没有连接、非正忙的实例
                        2、查找未启动的实例
                        3、查找正在启动的实例
                        4、查找不限制连接数的

REST: /manage/instance/free

实例：设置实例运行参数 ↑↑↑

            
接口说明：
        给指定实例设置参数
        详细的错误代码请参考ManageResult

REST: PUT /manage/instance/update

数据格式：
        {
            quiet: true,
            async: false,   //async参数是可选的，如果设置为true，那么立即返回结果，如果设置为false或者没有此参数，会等待实例启动结果，然后再返回
            startup: true,  //是否启动实例
            staticInstance: {
                id: '2491836508538',
                adjustResolution: true,
                limitOneClient: true,
                locked: true,
                pauseWhenIdle: false,
                websocketPort: 4321
            }
        }


            

                

                    

                        实例ID
                        
                            
                        
                        
                    

                    

                        工程
                        
                        
                            
                                如果工程在CloudMaster的工程列表里，此处可以输入acp全路径或者工程名，
                                如果不在列表里，此处必须是全路径。 另外此处也可以输入工程ID（即pid），
                                如果输入pid，实际设置的参数是实例的projectId属性
                        
                    

                    

                        
                        
                            
                            
                        
                        
                             如果勾选则实例将不会显示窗口，如果没有勾选，则不会设置quiet属性，至于是在前台还是在后台运行由NodeService决定
                        
                    

                    

                        
                        
                            
                            
                        
                        
                            如果勾选，WebSocket会等待实例启动结果，然后再返回
                        
                    

                    

                        
                        
                            
                            
                        
                        
                            如果勾选，设置参数后将会启动该实例。 如果勾选了锁定工程文件，可以不用再设置此属性，因为锁定工程肯定会自动启动实例
                        
                    

                    

                        
                        
                            
                        
                        
                            设置此参数后，如果实例未运行则会像startup参数一样启动实例；如果实例正在运行，则会重启实例
                        
                    

                    

                        
                        
                            
                        
                        
                            锁定工程文件后，该实例就只能使用指定的工程文件，客户端访问时不能再通过pid参数动态设置成别的工程文件了
                        
                    

                    

                         
                        
                            
                        
                        
                    

                    

                        
                        
                            
                        
                        
                    

                    

                        
                        
                            
                        
                        
                    

                    

                        WebSocket端口
                        
                            
                        
                        
                            如果设置为0，则不开启WebSocket调用方式
